## 2024-05-24 - Missing CSRF Protection on State-Modifying Admin Routes
**Vulnerability:** Found state-modifying API routes (POST, PATCH, DELETE) in the Next.js dashboard, specifically in `dashboard/src/app/api/admin/server/danger/route.ts`, that were missing CSRF protection.
**Learning:** State-modifying API routes in Next.js using session cookies must be protected against Cross-Site Request Forgery (CSRF). While Next.js 13+ App Router provides some default CSRF protections for Server Actions, API routes (`route.ts`) still require explicit CSRF checks when using cookie-based auth (like Clerk sessions). The codebase already has a helper function `verifyCsrf` in `@/lib/auth-permissions`, but it was not being consistently applied to all sensitive endpoints, leaving server ownership transfer and server deletion actions vulnerable.
**Prevention:** Always use the `verifyCsrf(req)` helper from `@/lib/auth-permissions` at the very beginning of all state-modifying API route handlers (POST, PATCH, DELETE) in the Next.js dashboard before performing any sensitive actions or session checks. Requests failing this check should immediately return a 403 Forbidden response.
