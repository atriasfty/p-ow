## 2024-06-03 - [CRITICAL] Fixed missing CSRF protection on danger API endpoints
**Vulnerability:** The `PATCH` and `DELETE` endpoints at `dashboard/src/app/api/admin/server/danger/route.ts` used for critical server owner actions (transfer ownership, change API key, delete server) lacked Cross-Site Request Forgery (CSRF) protection. Next.js App Router API routes (`route.ts`) do not have automatic CSRF protection when using cookie-based sessions.
**Learning:** State-modifying API routes using cookie-based authentication must explicitly implement CSRF validation. Without this, an attacker could trick an authenticated server owner into executing a malicious script on a third-party site that submits requests to these endpoints, potentially hijacking or destroying the server.
**Prevention:** Always use the `verifyCsrf(req)` helper from `@/lib/auth-permissions` at the very beginning of state-modifying handlers (POST, PATCH, DELETE) to reject unauthorized cross-origin requests with a 403 response before processing any sensitive logic.
